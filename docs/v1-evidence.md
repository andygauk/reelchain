# ReelChain v1: From Cast-Lookup Toy to a Deployed Puzzle Game

*How a weekend idea turned into a self-hosted, auto-deploying film-to-film actor-bridge game — and the key activities that got v1 live.*

---

## What is ReelChain?

ReelChain is a browser puzzle. You're given two films and a challenge: **connect them through shared actors**, clicking a performer they have in common, then a film *that* actor was in, then the next actor — building the shortest possible chain.

Example path: *The Godfather* → **Al Pacino** → *Once Upon a Time in Hollywood* → **Tim Roth** → *Pulp Fiction*.

It's "Six Degrees of Kevin Bacon" you actually *play* — the graph is real (118 films, 340 actors baked in at build time), the shortest-path solving is live, and every win is verified.

**Play it:** https://reelchain.app
**Source:** https://github.com/andygauk/reelchain

## The journey to v1

### 1. From lookup tool to playable game
It started as a static cast-lookup page — pick a film, see its actors. That was fine, but the interesting question was never "who's in this film?" it was "how do two films connect?" So it was rebuilt as an interactive actor-bridge puzzle: a single self-contained HTML file with the graph embedded, no backend, no API, no database.

### 2. One source of truth
The whole thing is generated by `build_game.py` from `data.json`. One command regenerates both the localhost game (`reelchain_game.html`) and the exact file Cloudflare serves (`deploy/index.html`). No drift between what you test and what's live.

### 3. The win-overlay bug (and the real fix)
The first "you won!" overlay rendered fine in standalone use but got clobbered by host-page CSS when embedded. The tempting fix — `!important` + reparenting the overlay to `<body>` — actually **broke the localhost version**, so it was reverted. The clean fix: namespace every class under `rc-`, nest the overlay *inside* `.rc-app`, keep `.rc-btn` unscoped with its own font/colour resets, and cap `z-index` at the max integer. One scoped island that can't be touched by anything around it.

### 4. Killing Daily mode
An early build had both a Daily Challenge and an Endless mode. The single daily game added friction with no payoff, so Daily was removed entirely — Endless with an easy/medium/hard difficulty selector is the only mode. Simpler to play, simpler to reason about.

### 5. WordPress — built, then removed
We did package it as a WordPress plugin (a `[reelchain]` shortcode, a `reelchain_game()` template tag, plus a Custom-HTML-block embed fragment) to drop into a site. But the deployment story moved to a static host, so all the WP artifacts were deleted — and, crucially, **stripped from the generator** so a rebuild can't silently recreate them. The repo is now WP-free by construction.

### 6. GitHub + Cloudflare auto-deploy
The repo landed on GitHub (`andygauk/reelchain`, `main`). Cloudflare Pages was connected to that repo: every `git push` to `main` now builds and deploys `deploy/` to the live URL automatically. Three-step loop: **edit locally → push → live.** No manual deploy commands, no pasted tokens.

## Try it

Open **https://reelchain.app**, pick a difficulty, and see how few hops you need. The graph is small on purpose — it's a tight, solvable puzzle, not a filmography.

## What's next (v2)

The graph is the obvious lever: more films and actors mean more surprising chains. Beyond that, a shareable result card, a streak/scoreboard, and maybe a "hardest pair" generator. v1 proved the loop works end-to-end — v2 is about making it stickier.
