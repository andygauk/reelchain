Quick wins you can ship today (CSS only)

Actor labels — 2 lines

  • .rc-nm { font-size: 13px; color: var(--text); max-width: 80px; white-space: normal; }
  • Fixes WCAG AA + readability on all viewports

Reduced motion — 1 @media block

  • Disable 13 animations for prefers-reduced-motion: reduce
  • Confetti, Linked! pop, gate pulse, chain dash all respect OS setting

Score details button — native details upgrade

  • Chevron, button styling, hover/focus, single-column grouped list
  • Works without JS; 1 line to sync aria-expanded

Focus ring — 1 rule

  • .rc-actor:focus-visible, .rc-film:focus-visible, .rc-ct-tile:focus-visible { outline: 2px solid var(--gold); outline-offset: 2px; }
  • Pair with JS enhancement for tabindex/role

Accessibility baseline after fixes

✅ Keyboard

  • Tab order: Start → Target → Actors (L→R) → Films (L→R)
  • Enter/Space triggers selection at every step
  • Focus visible everywhere (gold 2px ring)

✅ Screen readers

  • All tiles: role="button" + aria-label ("Choose Leonardo DiCaprio")
  • Start/Target: "Start film: X" / "Target film: Y"
  • Win screen: proper heading hierarchy (h1 → h2)
  • Score details: native disclosure announced correctly

✅ Motion

  • prefers-reduced-motion respected globally
  • Confetti hidden, Linked! instant, gate glow static

✅ Visual

  • Actor names 13px / high contrast / wrapped
  • Instruction adjacent to actor deck
  • Target card has equal visual presence
  • All contrast ratios ≥ 4.5:1 (measured: 5.85–16.18:1)

✅ Mobile

  • Actor deck scrolls horizontally (already works)
  • Score details single-column list on < 480px
  • Tap targets ≥ 44px (actors 62×68, films 100×150, buttons 41px tall)
Method: Live audit via browser console (computed styles, contrast math, DOM inspection) + browser_vision screenshots. No code changes made — sketches are standalone HTML in sketches/ux-review/. Open any index.html in a browser to interact.
